# Midas Core - Financial Transaction Processing System

## 🏦 Project Overview
Enterprise-grade financial transaction processing system developed during JPMorgan Chase Software Engineering Virtual Experience Program.

This project demonstrates building scalable backend systems for high-volume financial operations using modern enterprise technologies.

---

## 🛠️ Technologies Used

- Spring Boot - Application framework
- Apache Kafka - Message queue for asynchronous processing
- SQL Database (H2) - Transaction validation and storage
- REST APIs - External system integration
- JPA/ORM - Database abstraction layer
- Git - Version control

---

## ✨ Key Features

Kafka Message Queue Integration
- Decoupled frontend and backend architecture
- Asynchronous transaction processing
- Scalable multiple producer/consumer support

SQL Database Implementation
- Robust transaction validation
- Data integrity for financial operations
- JPA-based database abstraction

REST API Integration
- Seamless external system communication
- Incentive API integration
- Request/response pattern implementation

User Account Management
- Real-time balance queries
- REST controller endpoints
- Secure data exposure

---

## 🏗️ System Architecture

Component Flow:
- Frontend sends transaction to Kafka Queue
- Midas Core processes transaction
- Validates against SQL Database
- Calls Incentive REST API
- Updates user balance
- Returns response to user

Key Benefits:
- Decoupling: Independent component modification
- Scalability: Easy horizontal scaling
- Reliability: Robust error handling for financial data
